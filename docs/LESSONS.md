## 1. Мета-уроки (процесс работы)

### 1.1. Правило self-improve (от PM, главное)

> «Анализируй все ошибки, анализируй причину и давай себе инструкции, чтобы в будущем они не повторялись. Делай это при каждом проходе.»

**Применяется как:** правило self-improve при любой ошибке.
1. Что случилось? (симптом)
2. Почему? (root cause, не симптом)
3. Какой класс? (синтаксис, логика, окружение, коммуникация)
4. Как предотвратить? (правило/паттерн/тест)
5. Записать в memory / KB.

### 1.2. Council = multi-perspective, не group-think

Использовал 5 экспертов (Contrarian, First Principles, Expansionist, Outsider, Executor). Главные победы:
- Поймал math error: «$1/day=$1M» оказалось 755к₽
- Вскрыл конфликт интересов блогера (платный клуб = bias)
- Перевернул цель 10М₽/мес → 10М₽/год с B2B-affiliate
- Убил Premium tier (999₽) — математика не сходилась

**Урок:** council — это 5 разных углов. **Противоречия между экспертами — feature, не bug.**

### 1.3. Не обещай того, что не проверил

Перед числом спрашивай «откуда это?». Если нет публичного источника — пиши «оценка ±50%» или «не найдено».

### 1.4. ИП ≠ ООО ≠ физлицо

Перед pitch-деком уточнить юр. форму. Шаблон:
- ИП/самозанятый → **займ** (+ опцион на будущую долю)
- ООО → **продажа доли** (5-25%)
- Фонд → **SAFE / конвертируемый займ** (12-18 мес согласований)

### 1.5. Работа в чате с non-technical PM

PM прямо сказал: «я не очень знающий в этом вопросе и могу что-то напутать». Это **сделало мой подход лучше**:
- Пошаговые инструкции (1 команда за раз)
- Объяснять **почему** каждая команда
- Указывать **ожидаемый вывод** после каждой команды
- Не использовать сокращения
- Копировать вывод терминала дословно в инструкции

## 2. BASH / DEPLOY ГРАБЛИ (главный раздел)

### 2.1. ❌ heredoc + `curl ... | bash`

**Симптом:** скрипт не выполняется или работает частично.

**Root cause:** bash читает stdin. Когда stdin — сам скрипт (через pipe), heredoc `<<'EOF'` начинает «пожирать» следующие строки скрипта, никогда не видит EOF.

**Решение:** `curl URL > file.sh && bash file.sh` — сохранить на диск, потом запустить.

### 2.2. ❌ sys.path.insert + импорт `from package.module`

**Симптом:** `ModuleNotFoundError: No module named 'X'`.

**Решение:** добавлять **родителя** пакета в `sys.path`, не сам пакет:
```python
sys.path.insert(0, '/opt/beget/PARENT')
from package.module import ...
```

### 2.3. ❌ `pkill -9 -f` ненадёжен

**Симптом:** Flask на 8080, pkill не убивает, новый не стартует.

**Решение:** искать PID по порту через `ss`/`lsof`/`fuser`. Или — **использовать systemd** (правильный паттерн для production).

### 2.4. ❌ Патчинг в конец файла с `if __name__ == '__main__'`

**Симптом:** Flask стартует, отвечает, но **твой код не выполняется** (роуты не регистрируются).

**Root cause:** `app.run()` блокирует главный поток. Код ПОСЛЕ `if __name__` никогда не запустится.

**Решение:** вставлять патч **ПЕРЕД** `if __name__ == '__main__':`.

**Самый коварный баг в деплое — потратили 30 минут на диагностику.**

### 2.5. ❌ Дублирование патча при повторных запусках

**Симптом:** после N запусков patch-скрипта в файле N копий одного блока.

**Решение:** Python regex для удаления ВСЕХ блоков по общему паттерну.

### 2.6. ❌ Забыл удалить `__pycache__/`

**Симптом:** Flask использует старый код, несмотря на изменения.

**Решение:** `rm -rf __pycache__/` перед стартом.

### 2.7. ❌ Терминал ест переводы строк при копипасте

**Симптом:** 4 команды склеиваются в одну.

**Решение:** давать **одну команду за раз**.

### 2.8. ❌ Git push с credentials в URL

**Симптом:** токен утекает в `git log`, `git remote -v`, `.git/config`.

**Решение:** `gh auth login` (CLI) или SSH-ключ. **Никогда** `https://TOKEN@github.com`.

### 2.9. ✅ Универсальный паттерн: регистрация Flask endpoints из внешнего пакета

```python
# В основном app-файле добавить ПЕРЕД 'if __name__':
import sys
sys.path.insert(0, '/opt/beget/PARENT_DIR_OF_PACKAGE')
try:
    from package_name.routes import register_routes
    register_routes(app)
except Exception as e:
    print(f"[PACKAGE] FAILED: {e}", flush=True)
    import traceback; traceback.print_exc()
```

**Всегда try/except + print traceback** — иначе Flask стартует молча, и ищешь проблему часами.

### 2.10. ✅ Чеклист: рестарт Flask

```bash
# 1. Найти PID по порту
PIDS=$(ss -tlnp 2>/dev/null | grep ':PORT' | grep -oP 'pid=\K[0-9]+')
# 2. Убить (НЕ pkill -9 -f)
[ -n "$PIDS" ] && kill -TERM $PIDS
# 3. Подождать
sleep 3
# 4. Удалить кэш
rm -rf /path/to/__pycache__
# 5. Проверить порт свободен
ss -tln | grep -q ':PORT' && exit 1
# 6. Стартовать с env
cd /path/to/app && ENV1=... nohup python3 app.py > app.log 2>&1 &
# 7. Проверить
sleep 4 && kill -0 $! && echo "OK"
# 8. Smoke test
curl -s http://localhost:PORT/health
```

**Или лучше:** используй systemd unit (см. `research-agent/deploy/newton-api.service`).

## 3. Архитектурные уроки

### 3.1. F&F-инвестиция ≠ венчурное финансирование

Разные суммы, доли, сроки, юр. формы, отчётность. Не натягивать VC-структуру на F&F.

### 3.2. Hard triggers зависят от существующей инфраструктуры

Сначала узнать, что уже есть (Postgres в Docker, Redis, etc.), потом писать triggers. Миграция = новые таблицы в той же базе, не переезд.

### 3.3. Constraint рождает фокус

Меньше workflows = проще отлаживать + меньше RAM. 6 workflows лучше 11.

### 3.4. Поведенческие механики (streak + freeze)

«Делай 30 дней подряд» проигрывает «streak + 1 freeze в квартал» (Duolingo, Apple Watch).

### 3.5. Inline-fraud > /fraud команда

Forward любого текста → бот проверяет. Убирает friction.

## 4. Unit-экономика уроки

### 4.1. LTV/CAC считать по каналам, не blended

Organic: 8-16x · Referral: 6-12x · Content: 3-6x · Paid: 1-2x.

### 4.2. Churn benchmarks для РФ consumer SaaS

5-8% — отлично, 12% — норма для первого года, 15%+ — продукт мёртв.

### 4.3. Goal realism: 10М₽/мес ≠ 10М₽/год

Council v1: 10М₽ MRR = 40к paid = невозможно без B2B. Реалистично: 250-550к MRR к Dec 2027, 800к-1.5М к Dec 2028.

### 4.4. Burn peaks = hire time

Каждый hire = 3x rule (revenue > 3x salary) или не нанимай.

### 4.5. Только public data, never invent

При отсутствии прямых данных — «оценка ±50%» или «не найдено в публичных источниках». Не выдумывай.

## 7. Главная мысль

**Когда работаешь над длинным проектом, самое ценное — не «сделать», а «сделать так, чтобы через 6 месяцев PM или новый агент мог разобраться и продолжить».**
